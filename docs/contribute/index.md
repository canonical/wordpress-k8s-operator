# Contribute

The WordPress Operator is a member of the Ubuntu family. It's an open-source
project that warmly welcomes community projects, contributions, suggestions,
fixes, and constructive feedback.

Contributing to this project provides an opportunity to grow your technical
skills, develop as a professional, and become involved in an open-source project. 

All contributors should become familiar with these guides. They outline the
expectations and practices for participating in the project.

## Standards and expectations

The Platform Engineering team at Canonical sets the direction and priorities
of the WordPress Operator. 
Please consider the following documents before contributing to this project.

### Ubuntu Code of Conduct

When contributing, you must abide by the
[Ubuntu Code of Conduct](https://ubuntu.com/community/ethos/code-of-conduct).

### Canonical Contributor License Agreement

To improve contribution tracking,
we use the [Canonical contributor license agreement](https://assets.ubuntu.com/v1/ff2478d1-Canonical-HA-CLA-ANY-I_v1.2.pdf)
(CLA) as a legal sign-off.

Please check out the [contributor agreement](https://ubuntu.com/legal/contributors),
and sign the agreement before preparing your contribution.

### Signing commits

To improve contribution tracking, we use the CLA as a legal
sign-off, and we require all commits to have verified signatures.

#### Canonical contributor agreement

The CLA sign-off is simple line at the
end of the commit message certifying that you wrote it
or have the right to commit it as an open-source contribution.

#### Verified signatures on commits

All commits in a pull request must have cryptographic (verified) signatures.
To add signatures on your commits, follow the
[GitHub documentation](https://docs.github.com/en/authentication/managing-commit-signature-verification/signing-commits).
